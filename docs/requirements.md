# Element Picker Requirements

## 目的

Element Picker は、Webページ上の任意のHTML要素を視覚的に選択し、その要素以下の内容を任意の形式でクリップボードにコピーできるブラウザ拡張機能である。

開発者ツールを開かなくても、ブラウザ標準の要素選択に近い操作感で、必要な範囲だけを素早く取り出せることを目的とする。

## 想定ユーザー

- Webページの一部をHTMLまたはMarkdownとして再利用したいユーザー
- 記事、ドキュメント、UI部品、表、カードなど、ページ内の特定範囲だけをコピーしたいユーザー
- AIツール、ノート、CMS、ドキュメント作成ツールへWebコンテンツを渡したいユーザー
- 開発者ツールを使えるが、コピー作業をより短い操作で済ませたいユーザー

## 基本コンセプト

- 拡張機能のアイコンをクリックすると、ページ上の要素選択モードに入る。
- ユーザーはページ上の要素をホバーしてコピー対象を確認できる。
- 要素をクリックすると対象が確定し、コピー形式を選択できる。
- 選択した形式で、対象要素以下の内容がクリップボードにコピーされる。
- コピー完了後は、要素選択モードを終了する。

## 主要ユーザーフロー

1. ユーザーがブラウザ拡張機能のアイコンをクリックする。
2. 現在開いているページが要素選択モードになる。
3. マウスカーソルを重ねた要素が視覚的に強調表示される。
4. ユーザーがコピーしたい要素をクリックする。
5. コピー形式の選択UIが表示される。
6. ユーザーがコピー形式を選ぶ。
7. 選択した要素以下の内容がクリップボードにコピーされる。
8. コピー完了がユーザーに分かる形で通知される。

## 必須要件

### 要素選択

- ページ上のHTML要素をマウス操作で選択できること。
- ホバー中の要素が明確に分かること。
- クリックした要素をコピー対象として確定できること。
- 選択モード中に誤ってページ本来のリンクやボタンが実行されないこと。
- 選択モードをキャンセルできること。

### コピー形式

初期バージョンでは、以下の形式に対応する。

- HTML
- Markdown
- プレーンテキスト

将来的な対応候補として、以下の形式を検討する。

- 画像
- JSON
- CSS Selector
- XPath

### HTMLコピー

- 選択した要素を含むHTMLをコピーできること。
- 選択した要素の内側だけをコピーする選択肢も検討できること。
- ページ全体ではなく、選択した要素以下に範囲を限定すること。

### Markdownコピー

- 選択した要素以下の内容をMarkdownとしてコピーできること。
- 見出し、段落、リスト、リンク、画像、表など、一般的な文書構造が可能な限り自然なMarkdownになること。
- 装飾やレイアウトよりも、再利用しやすい本文構造を優先すること。

### テキストコピー

- 選択した要素以下の可読テキストをコピーできること。
- ユーザーが画面上で読める内容に近いテキストになること。
- 不要なHTMLタグや属性を含めないこと。

### 画像コピー

画像コピーは初期必須要件ではなく、将来対応候補とする。

対応する場合は、以下を満たすことを目指す。

- 選択した要素の見た目を画像としてコピーまたは保存できること。
- 画面上の表示に近い結果になること。
- 画像化できない要素や外部制約がある場合は、失敗理由が分かること。

## UI要件

- 要素選択中であることが明確に分かること。
- ホバー中の要素は、ページ内容を大きく邪魔しない形で強調表示すること。
- コピー形式の選択UIは、選択した要素の近く、または操作しやすい位置に表示すること。
- コピー形式は短いラベルで分かりやすく表示すること。
- コピー完了、キャンセル、失敗がユーザーに分かること。
- ページ本来の見た目や操作を、選択モード終了後に残さないこと。

## 操作要件

- 拡張機能アイコンのクリックで開始できること。
- クリックで要素を確定できること。
- キャンセル操作を用意すること。
- コピー形式選択後は自動的に通常状態へ戻ること。
- 連続して別の要素をコピーする操作も将来的に検討できること。

## 品質要件

- コピー対象の範囲がユーザーの選択と一致していること。
- 操作が軽く、通常の閲覧を妨げないこと。
- コピー結果が予測しやすいこと。
- ページに一時的な表示を追加しても、操作終了後に残らないこと。
- 一般的なWebページ、記事ページ、ドキュメントページ、管理画面、表形式のページで使えること。

## エラー・制約

- ブラウザやページの制約によりコピーできない場合は、失敗したことをユーザーに伝えること。
- 一部のページ、埋め込みコンテンツ、特殊な表示要素では期待通りに取得できない可能性がある。
- 画像コピーは、ページ内容や外部リソースの条件により結果が不完全になる可能性がある。
- パスワード、トークン、個人情報などを意図せずコピーしないよう、ユーザーが選択範囲を確認できることを重視する。

## 初期リリース範囲

初期リリースでは、以下を対象とする。

- 拡張機能アイコンから要素選択モードを開始する。
- ホバー中の要素を強調表示する。
- クリックで要素を確定する。
- HTML、Markdown、プレーンテキストのいずれかでコピーする。
- コピー完了または失敗を通知する。
- 選択モードをキャンセルできる。

初期リリースでは、以下は必須対象外とする。

- 画像コピー
- JSONコピー
- CSS Selectorコピー
- XPathコピー
- 複数要素の同時選択
- ページをまたいだ自動収集
- コピー履歴管理
- クラウド同期

## 将来検討項目

- 画像としてコピーまたは保存する機能
- CSS SelectorやXPathのコピー
- HTML、Markdown、テキストを同時に含むJSONコピー
- コピー形式の既定値設定
- ショートカットキーによる起動
- 連続コピー用の固定選択モード
- 選択範囲の親要素・子要素への移動
- コピー前プレビュー
- 不要要素を除外してコピーする機能
- iframe内の要素選択
- Shadow DOMを含む要素選択

## 非目標

- Webスクレイピングツールとして大量データを収集すること
- ページ全体を保存・アーカイブすること
- 開発者ツールを置き換えること
- ページの内容を編集すること
- 自動で最適な本文範囲を推測して抽出すること
